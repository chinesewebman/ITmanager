## 三、机柜图能力分析（关键发现）

### Zabbix
- ✅ 网络地图 (Network Maps)
- ✅ 自定义仪表盘 (Custom Dashboards)
- ❌ **无内置机架视图**

### LibreNMS
- ✅ 网络地图 (Availability Map, Dependency Map)
- ❌ **无内置机架视图**

### 结论
**Zabbix 和 LibreNMS 都没有开箱即用的机柜图功能**，无论选择哪个监控层，机柜图都需要自建或依赖第三方工具。

## 四、最终选型方案：NetBox + Zabbix

### 4.1 方案概述

经过调研确定采用 **NetBox + Zabbix** 组合方案：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    选型方案架构                                       │
├─────────────────────────────────────────────────────────────────────┤

│  │   NetBox    │    │    Zabbix    │    │  自建平台    │    │
│  │  (机柜图)   │◄──►│   (监控)     │    │  (业务层)    │    │

│        │                     │                     │             │
```

### 4.2 NetBox 功能

| 功能 | 说明 |
|------|------|
| **Rack View** | 可视化机架设备摆放 |
| **Cable Trace** | 物理连接追踪 |
| **IPAM** | IP地址管理 |
| **DCIM** | 数据中心基础设施管理 |
| **REST API** | 完整的API接口 |
| **Webhook** | 事件触发通知 |

### 4.3 Zabbix 功能

| 功能 | 说明 |
|------|------|
| **SNMP监控** | 支持 v1/v2c/v3 |
| **自动发现** | 网络发现、Agent注册 |
| **模板生态** | 1000+官方+社区模板 |
| **告警机制** | 灵活的告警规则 |
| **API** | 完整的API接口 |

### 4.4 集成方式

1. **NetBox → Zabbix**
   - 通过 NetBox Webhook 触发 Zabbix API 创建监控项
   - 或通过自定义脚本同步设备到 Zabbix

2. **Zabbix → NetBox**
   - Zabbix 告警时调用 NetBox API 更新设备状态
   - 自动关联设备位置信息

3. **自建平台整合**
   - NetBox 作为资产源数据 (Source of Truth)
   - Zabbix 作为监控数据源
   - 自建平台调用双方 API 呈现统一视图

## 五、机柜图交互设计（需求细化）

### 5.1 基础展示

- **机柜视图**：默认只显示设备状态指示灯
  - 🟢 绿灯：设备正常
  - 🟡 黄灯：存在警告
  - 🔴 红灯：告警/离线

### 5.2 鼠标悬浮交互

- **悬浮显示**：鼠标移动到设备上时，浮现主要指标
  - CPU 使用率图表
  - 内存使用率图表
  - **告警相关指标**（高亮显示）

### 5.3 点击跳转

- **点击操作**：点击设备后弹出详情页
  - 跳转到 Zabbix 完整监控页面
  - 显示历史告警记录
  - 显示性能趋势图
