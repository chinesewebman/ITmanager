### 7.2 组件选型汇总

| 层级 | 组件 | 选型 | 说明 |
|------|------|------|------|
| **资产管理** | NetBox | ⭐ 推荐 | DCIM/IPAM/机柜图 |
| **监控告警** | Zabbix | ⭐ 推荐 | SNMP/Agent监控 |
| **运维工单** | GLPI | ⭐ 推荐 | ITIL工单系统 |
| **日志管理** | Graylog | 可选 | 日志集中管理 |
| **备份** | BorgBackup | 可选 | 增量备份 |

### 7.5 模块设计

#### 7.5.1 资产服务 (Asset Service)

```
职责：资产管理全生命周期

核心API:
- POST   /api/v1/assets              创建资产
- GET    /api/v1/assets              查询资产列表
- GET    /api/v1/assets/:id          查询资产详情
- PUT    /api/v1/assets/:id          更新资产
- DELETE /api/v1/assets/:id          删除资产
- GET    /api/v1/racks               机柜列表
- GET    /api/v1/racks/:id/devices   机柜设备

数据来源:
- NetBox API (主数据源)
- Zabbix (监控状态)
- 手动录入
```

#### 7.5.2 告警服务 (Alert Service)

```
职责：告警数据聚合、展示

核心API:
- GET    /api/v1/alerts              告警列表
- GET    /api/v1/alerts/:id          告警详情
- PUT    /api/v1/alerts/:id/ack      确认告警
- PUT    /api/v1/alerts/:id/resolve  解决告警
- GET    /api/v1/alerts/stats        告警统计

数据来源:
- Zabbix API
```

#### 7.5.3 工单服务 (Ticket Service)

```
职责：工单数据聚合、展示

核心API:
- GET    /api/v1/tickets             工单列表
- GET    /api/v1/tickets/:id        工单详情
- POST   /api/v1/tickets             创建工单

数据来源:
- GLPI API
```

### 7.6 系统规模适配

针对 **3机房、几十机柜、1000+VM、几十台物理机** 规模的建议：

| 组件 | 配置建议 |
|------|----------|
| **NetBox** | 1台 4C8G |
| **Zabbix** | 1台 8C16G (含PostgreSQL) |
| **GLPI** | 1台 4C8G |
| **Graylog** | 可选，1台 4C8G |
| **自建API** | 1-2台 2C4G |
