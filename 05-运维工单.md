## 五、运维工单详细设计

### 5.1 工单系统选型：GLPI

#### 5.1.1 GLPI 优势

| 特性 | 说明 |
|------|------|
| **ITIL兼容** | 符合ITIL最佳实践 |
| **资产管理** | 内置资产CMDB |
| **工单管理** | 完整的请求/故障/变更流程 |
| **API** | REST API支持 |
| **插件生态** | 丰富插件扩展 |
| **多组织** | 支持多部门/多客户 |

#### 5.1.2 GLPI 组件

| 组件 | 说明 |
|------|------|
| Web界面 | PHP + MySQL/MariaDB |
| 定时任务 | cron 任务 |
| 邮件收件 | 邮件网关 |

### 5.2 工单类型

| 类型 | 说明 | 触发方式 |
|------|------|----------|
| **请求 (Request)** | 服务请求、配置变更 | 手动/监控触发 |
| **故障 (Incident)** | 设备故障、服务中断 | 手动/监控触发 |
| **变更 (Change)** | 计划变更、配置调整 | 手动审批 |
| **问题 (Problem)** | 根因分析 | 手动创建 |

#### 5.5.2 GLPI API 调用示例

```python
import requests

GLPI_URL = "https://glpi.company.com/apirest.php"
GLPI_APP_TOKEN = "your-app-token"

def create_glpi_ticket(alert_data):
    # 获取 Session Token
    session = requests.post(
        f"{GLPI_URL}/initSession",
        headers={"App-Token": GLPI_APP_TOKEN},
        data={"user_token": "user-api-token"}
    ).json()["session_token"]

    # 创建工单
    ticket = requests.post(
        f"{GLPI_URL}/Ticket/",
        headers={
            "App-Token": GLPI_APP_TOKEN,
            "Session-Token": session,
            "Content-Type": "application/json"
        },
        json={
            "name": f"Zabbix告警: {alert_data['hostname']}",
            "content": alert_data['message'],
            "itilcategories_id": 1,  # 监控告警分类
            "urgency": map_urgency(alert_data['severity']),
            "impact": map_impact(alert_data['severity']),
            "priority": calculate_priority(alert_data['severity'])
        }
    ).json()

    # 关闭 Session
    requests.get(f"{GLPI_URL}/killSession", headers={"Session-Token": session})

    return ticket
```

#### 5.5.3 工单字段映射

| Zabbix告警 | GLPI工单字段 |
|------------|--------------|
| 告警名称 | name (标题) |
| 告警消息 | content (内容) |
| 告警级别 | urgency/priority |
| 主机名 | items_id (关联资产) |
| 触发时间 | time_to_resolve (SLA) |
