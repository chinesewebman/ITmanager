# 网络运维监控平台 - 监控采集模块

## 三、监控采集详细设计

### 3.1 监控方案选型：Zabbix

#### 3.1.1 Zabbix 优势

| 特性 | 说明 | 适用场景 |
|------|------|----------|
| **SNMP支持** | v1/v2c/v3 完整支持 | 网络设备监控 |
| **Agent支持** | Zabbix Agent / Agent 2 | 服务器监控 |
| **模板生态** | 1000+官方+社区模板 | 快速接入设备 |
| **自动发现** | 网络发现、主动注册 | 批量添加设备 |
| **告警机制** | 灵活阈值、升级、抑制 | 告警管理 |
| **API** | 完整 REST API | 集成自动化 |

#### 3.1.2 Zabbix 组件

| 组件 | 数量 | 说明 |
|------|------|------|
| Zabbix Server | 1-2 | 核心服务，支持HA |
| Zabbix Web | 1 | Web界面 |
| Zabbix Agent | N | 部署在被监控服务器 |
| PostgreSQL | 1 | 数据库存储 |

### 3.2 监控指标体系

#### 3.2.1 指标类别

| 指标类别 | 采集方式 | 示例 |
|----------|----------|------|
| **系统指标** | Agent | CPU、内存、磁盘、网络 |
| **网络指标** | SNMP | 接口流量、延迟、丢包 |
| **应用指标** | Agent/API | 进程、端口、服务状态 |
| **硬件指标** | IPMI/ SNMP | 温度、电源、风扇、RAID |
| **VMware指标** | vSphere API | VM资源、ESXi状态 |

#### 3.2.2 关键监控指标

**服务器 (Linux/Windows):**
- CPU使用率、负载、上下文切换
- 内存使用、交换分区
- 磁盘使用、I/O、inode
- 网络接口流量、连接数
- 进程数、端口监听
- 服务状态

**网络设备 (交换机/路由器/防火墙):**
- 接口状态、流量 (in/out)
- CPU/内存使用率
- 端口速率、错误统计
- VLAN信息
- 硬件状态 (电源/风扇)

**VMware 虚拟化:**
- ESXi 主机状态
- VM 运行状态
- CPU/内存使用
- 数据存储状态
- VM 迁移状态

**常用 OID:**

| OID | 名称 | 适用设备 |
|-----|------|----------|
| .1.3.6.1.2.1.1.1.0 | sysDescr | 所有设备 |
| .1.3.6.1.2.1.1.3.0 | sysUpTime | 所有设备 |
| .1.3.6.1.2.1.2.2.1.* | ifTable | 网络设备 |
| .1.3.6.1.2.1.25.1.1.0 | hrSystemUptime | 服务器 |

#### 3.3.2 Zabbix Agent采集

```yaml
# zabbix_agentd.conf 关键配置
Server=zabbix-server.example.com
ServerActive=zabbix-server.example.com
Hostname=server-01

# 监控项配置
UnsafeUserParameters=1
UserParameter=custom.cpu.stat[*],/scripts/cpu_stat.sh $1
```

#### 3.3.3 VMware采集

```python
# 使用 pyvmomi 采集 VMware 指标
from pyVmomi import vim

def get_vm_metrics(vm):
    summary = vm.summary
    return {
        "cpu_usage": summary.quickStats.cpuUsage,
        "memory_usage": summary.quickStats.memoryUsage,
        "status": summary.runtime.powerState
    }
```

### 3.5 采集任务调度

#### 3.5.1 调度策略

| 策略 | 间隔 | 适用场景 |
|------|------|----------|
| **常规采集** | 60秒 | CPU、内存等基础指标 |
