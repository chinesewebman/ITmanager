# 网络运维监控平台 - 安全合规设计

## 十、安全合规设计

### 10.1 SNMP通讯字安全检查

#### 10.1.1 合规标准

| 等级 | 通讯字类型 | 处理方式 |
|------|------------|----------|
| 🔴 **不合规** | public、private、manager、default、admin、root、12345 等常见默认字符串 | 记录并告警，要求整改 |
| 🟡 **警告** | 短于8位、纯数字、纯字母 | 建议修改，但不强制 |
| 🟢 **合规** | 复杂组合（大小写+数字+特殊字符，长度≥12位） | 正常通过 |

#### 10.1.2 合规检查流程

```
发现任务执行
     │
     ▼
遍历网段IP ──► SNMP探测（尝试通讯字列表）
     │
     ▼
  通讯成功？
     │
   ┌─┴─┐
   │   │
   是  否
   │   │
   ▼   ▼
检查通讯字 ◄────────── 下一IP
是否合规
   │
 ┌─┴─┐
 │   │
合规 不合规
 │   │
 ▼   ▼
标记"合规" 标记"不合规" + 生成整改工单
```

### 10.3 访问控制

#### 10.3.1 权限模型

采用 RBAC (基于角色的访问控制) + LDAP 组映射。

```
┌─────────────────────────────────────────────────────────────┐
│                      权限检查流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   用户 ──► LDAP验证 ──► 角色映射 ──► 权限检查            │
│           │                  │                              │
│           ▼                  ▼                              │
│       验证密码          分配权限                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 10.3.2 默认角色

| 角色 | 权限 |
|------|------|
| **管理员** | 完全控制 |
| **运维工程师** | 资产管理、监控配置、告警处理 |
| **只读用户** | 查看所有资源 |

### 10.4 安全审计

#### 10.4.1 审计日志内容

| 字段 | 说明 |
|------|------|
| timestamp | 操作时间 |
| user_id | 用户ID |
| username | 用户名 |
| action | 操作类型 |
| resource | 资源类型和ID |
| ip_address | 客户端IP |
| result | 操作结果 |

#### 10.4.2 审计日志存储

```

│                    审计日志存储                               │
```
