## 四、告警通知详细设计

### 4.1 告警方案：基于 Zabbix

#### 4.1.1 Zabbix 告警优势

| 特性 | 说明 |
|------|------|
| **阈值触发** | 支持多种运算符和函数 |
| **时间触发** | 支持时间周期配置 |
| **依赖触发** | 支持告警依赖关系 |
| **升级机制** | 支持告警升级策略 |
| **抑制机制** | 支持告警抑制/聚合 |
| **通知渠道** | 支持多种通知方式 |

#### 4.1.2 告警级别定义

| 级别 | 名称 | 颜色 | 描述 |
|------|------|------|------|
| ** Disaster** | 灾难 | 🔴 红色 | 系统不可用 |
| **High** | 高 | 🟠 橙色 | 严重故障 |
| **Average** | 中 | 🟡 黄色 | 服务降级 |
| **Warning** | 警告 | 🔵 蓝色 | 潜在问题 |
| **Information** | 信息 | 🟢 绿色 | 通知信息 |

### 4.2 通知渠道实现

#### 4.2.1 支持的渠道

| 渠道 | 类型 | Zabbix支持 | 适用场景 |
|------|------|------------|----------|
| **钉钉** | Webhook | ✅ 原生 | 实时告警 |
| **企业微信** | Webhook | ✅ 脚本 | 实时告警 |
| **邮件** | SMTP | ✅ 原生 | 日常通知 |
| **短信** | API | ✅ 脚本 | 紧急告警 |
| **电话** | API | ✅ 脚本 | P0级别告警 |
| **Webhook** | HTTP | ✅ 原生 | 第三方集成 |

#### 4.2.2 钉钉通知配置

```yaml
# Zabbix 钉钉告警脚本
# /usr/lib/zabbix/externalscripts/dingding.sh

#!/bin/bash

WEBHOOK_URL="https://oapi.dingtalk.com/robot/send?access_token=xxxxx"
SECRET="SECxxxxxxxx"

# 生成签名
timestamp=$(date +%s000)
sign=$(echo -n "${timestamp}\n${SECRET}" | openssl dgst -sha256 -hmac "${SECRET}" -binary | base64)
sign=$(python3 -c "import urllib.parse; print(urllib.parse.quote('$sign'))")

# 发送消息
curl -X POST "$WEBHOOK_URL&timestamp=$timestamp&sign=$sign" \
  -H 'Content-Type: application/json' \
  -d '{
    "msgtype": "markdown",
    "markdown": {
      "title": "Zabbix告警",
      "text": "### '${ALERT_SUBJECT}'\n\n"'"${ALERT_MESSAGE}"'""
    }
  }'
```

│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐     │

#### 4.3.2 告警状态

| 状态 | 说明 | 可转换到 |
|------|------|----------|
| **Problem** | 已触发，等待处理 | Acknowledged, Resolved |
| **Acknowledged** | 已确认 | Resolved |
| **Resolved** | 已解决 | Problem (复训) |

### 4.4 告警规则配置示例

#### 4.4.1 服务器CPU告警

```yaml
# Zabbix Trigger 配置
名称: Server CPU usage is too high
表达式: avg(/Linux server CPU/system.cpu.util,5m) > 80
严重性: Warning
描述: CPU使用率连续5分钟超过80%
```

#### 4.4.2 网络设备端口down

```黄山
名称: Network interface {ITEM.NAME} is down
表达式: nodata(/SNMP Generic/net.if.status[{#IFNAME}],10m) = 1 and last(/SNMP Generic/net.if.status[{#IFNAME}]) = 2
严重性: High
描述: 接口 {#IFNAME} 处于down状态
```

#### 4.4.3 虚拟机资源告警

```yaml
名称: VM {$VM.NAME} CPU usage high
表达式: avg(/VMware VM/vm.cpu.usage[{$VM.UUID}],5m) > {$VM.CPU.THRESHOLD}
严重性: Average
描述: 虚拟机CPU使用率超过阈值
```

### 4.5 告警与工单联动

#### 4.5.1 自动创建工单

```python
# Zabbix 告警回调脚本
# 自动创建运维工单

import requests

def create_ticket_from_alert(alert_data):
    # 调用 GLPI API 创建工单
    response = requests.post(
        "https://glpi.company.com/apirest.php/Ticket/",
        headers={
            "Authorization": f"AppToken {GLPI_APP_TOKEN}",
            "Session-Token": glpi_session
        },
        json={
            "name": alert_data['subject'],
            "content": alert_data['message'],
            "urgency": map_urgency(alert_data['severity']),
            "category": detect_category(alert_data['trigger']),
            "requesttypes_id": 1  # 监控告警
        }
    )
    return response.json()
```
