## 二十一、v2.0 路线图 (2026-06-17 决策)

详见 [ADR-0002](docs/adr/0002-v2-scope.md)。

### 21.1 v2.0 范围

| 模块 | 内容 | 估时 | 优先级 |
|------|------|------|--------|
| **cursor 分页** | alerts / audit_logs / tickets / notifications 翻页 O(log N) | 6-8h | P0 |
| **event bus** | in-process pub/sub, 通知解耦, dead-letter 队列 | 8-10h | P0 |
| **gRPC** | 内部 s2s 通信, REST 兼容 | 12-15h | P1 |
| **v2.0.2 压测 + release** | wrk/ab 压测, 性能 baseline | 4-6h | P1 |

### 21.2 明确不做 (v2.0 scope 外)

- ❌ Redis 部署 (v1.4 in-process LRU 够, 推迟 v3.0)
- ❌ Kafka/NATS (永久不做, 单机不需要)
- ❌ 多实例部署 (推迟 v3.0)
- ❌ 全文搜索 ES (推迟 v3.0)
- ❌ PWA 移动端 (推迟 v2.1)

### 21.3 兼容性原则

- v1.x 客户端继续用 `?page=N&size=M` (cursor=null 走 offset 兼容)
- DB schema 不变
- 部署方式不变 (单 binary + sqlite/postgres)
- 接口导向: cache 已有 interface, v3.0 切 Redis 零成本

### 21.4 验收标准

| 项 | 标准 |
|---|---|
| 翻页性能 | 100k 行 alerts 翻到末页 P95 < 200ms |
| 事件吞吐 | 单进程 > 1000 events/s, 无丢失 |
| 通知延迟 | event → 钉钉/邮件送达 P95 < 10s |
| gRPC 内部 RTT | < 5ms (localhost) |
| 测试覆盖 | service > 85%, total backend > 80% |
| tsc 错误 | 0 |
| 实测估时 | 30-40h → 实测 10-15h (按 v1.x 2.7x 加速) |

### 21.5 里程碑

| 版本 | 内容 | 累计估时 |
|------|------|----------|
| v2.0.0 | cursor 分页 + event bus | 14-18h |
| v2.0.1 | gRPC 内部通信 | 12-15h |
| v2.0.2 | 压测 + release | 4-6h |
