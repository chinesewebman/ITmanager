### 2.2 自动发现模块

#### 2.2.1 发现来源

| 来源类型 | 工具 | 说明 |
|----------|------|------|
| **SNMP扫描** | Zabbix | 通过SNMP协议扫描网段内的设备 |
| **Ping探测** | Zabbix / fping | ICMP探测存活主机 |
| **ARP扫描** | NetBox Discovery | 获取局域网内MAC地址和IP对应关系 |
| **VMware API** | VMware vSphere API | 自动发现虚拟化环境中的VM |
| **API导入** | 手动/脚本 | 第三方系统API导入 |

#### 2.2.2 跨网段整合

```
┌─────────────────────────────────────────────────────────────┐
│                     发现任务调度中心                         │
├─────────────────────────────────────────────────────────────┤
│  发现任务A (192.168.1.0/24)  ─┐                            │
│  发现任务B (10.0.0.0/8)       │──> 统一归一化处理 ──> NetBox │
│  发现任务C (172.16.0.0/12)  ─┘                            │
└─────────────────────────────────────────────────────────────┘
```

- **多任务并行**：不同网段可并行发起发现任务
- **统一归一化**：所有来源的数据统一格式、去重、关联
- **增量更新**：发现结果与已有资产比对，自动识别新增/变更/下线

### 2.5 集成接口设计

#### 2.5.1 NetBox API 调用

```python
# 示例：创建设备
import requests

NETBOX_URL = "http://netbox:8000"
NETBOX_TOKEN = "your-api-token"

def create_device_in_netbox(device_data):
    headers = {
        "Authorization": f"Token {NETBOX_TOKEN}",
        "Content-Type": "application/json"
    }
    response = requests.post(
        f"{NETBOX_URL}/api/dcim/devices/",
        json=device_data,
        headers=headers
    )
    return response.json()
```

#### 2.5.2 Zabbix API 调用

```python
# 示例：创建监控主机
def create_host_in_zabbix(host_data):
    response = requests.post(

            "method": "host.create",
            "params": host_data,
            "auth": zabbix_auth_token,
            "id": 1
        }
    )

```
